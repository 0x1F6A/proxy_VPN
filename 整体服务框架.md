# 企业级商用 VPN（机场）整体服务框架

> **项目代号**：proxy_VPN
> **定位**：面向中高端用户与企业的商用级多协议代理服务（"机场"），覆盖美国、日本等多地区 VPS 节点，套餐 + 流量包混合计费，支持支付宝 / 微信 / USDT(TRC20)。
> **文档版本**：v0.2 (2026-05-26) — 对齐 Phase 12 实施现状
> **维护者**：@0x1F6A
> **配套文档**：[`docs/architecture.md`](./docs/architecture.md)（工程深度展开）· [`docs/api.md`](./docs/api.md)（接口契约）· [`docs/ops.md`](./docs/ops.md)（运维）· [`docs/开发顺序与稳定性检查.md`](./docs/开发顺序与稳定性检查.md)

## 1. 产品定位

| 维度 | 内容 |
|------|------|
| **目标用户** | 海外留学/工作华人、跨境电商、自由职业、企业内网代理、技术发烧友 |

## 2. 技术栈（已落地）

### 2.1 后端
- **语言**：Go **1.25**（高并发、低内存、单二进制部署）
- **HTTP**：Gin（v1 / admin / 用户面 / 订阅 / 健康端点统一收口）
- **ORM**：GORM v2（MySQL 8）+ 显式迁移脚本（`internal/migrations`）
- **缓存/队列**：Redis 7 + **Asynq**（异步任务、定时任务、重试）
- **统计仓库**：ClickHouse（流量日志聚合、月表 / 物化视图）
- **可观测**：zerolog 结构化日志 + Prometheus 指标 + OpenTelemetry trace + Loki 集中日志
- **配置**：Viper + 环境变量覆盖（`PROXYVPN_*`）

### 2.2 代理核心（数据面）
- **Xray-core**：VLESS / VMess / Trojan / Shadowsocks / Shadowsocks-2022 / **Reality**
- **Sing-box**：作为多协议聚合层；统一配置抽象
- **Hysteria2 / TUIC v5**：基于 QUIC 的高速协议
- **传输**：TCP / WebSocket / gRPC / HTTPUpgrade / Reality / XHTTP
- **可选**：WireGuard 模式

### 2.3 存储
| 存储 | 用途 |
|------|------|
| **MySQL 8** | 业务主数据：用户、订单、支付、套餐、节点、流量包、优惠券、邀请、审计 |
| **Redis 7** | 缓存、会话黑名单、限流计数、Asynq 队列、邮件验证码 |
| **ClickHouse** | 流量日志、月度聚合、运营报表（按需启用） |
| **S3 / R2** | 备份、订阅托管、客户端分发包 |

### 2.4 前端（均已落地）
- **用户控制台** `web/user`：React 18 + TypeScript + **Vite** + Ant Design 5 + TanStack Query + Zustand
  - 页面：Login / Register / Dashboard / Plans / Orders / Pay / Subscription / Nodes / Invite / Profile / Help
- **运营后台** `web/admin`：技术栈同上
  - 页面（Phase 11 规划）：Dashboard / Users / Orders / Payments / Plans / Coupons / DataPacks / Nodes / NodeGroups / Reports
- 前端通过各自 `cmd/*-web` 二进制以 `go:embed dist` 静态托管，单一进程，无需独立 Nginx 服务前端

### 2.5 基础设施
- **容器**：Docker + Docker Compose（开发 / 小规模生产）；Helm Chart（K8s）
- **网关**：Caddy（自动 TLS、HTTP/3）+ Cloudflare CDN/WAF 前置
- **CI/CD**：GitHub Actions（lint / test / 集成测试 / gosec 安全扫描 / 多二进制并行发布 GHCR）
- **镜像**：Distroless 基础镜像，非 root 运行，多 stage 构建

**关键边界**
- **控制面 ↔ 数据面物理隔离**：控制域名（`api.example.com`）与节点域名（`jp01.cdn.example.com`）独立，不互暴露
- **进程拆分但共享 Bootstrap**：`cmd/*` 均复用 `internal/pkg` 的 Bootstrap，只在路由/中间件层差异化
- **横向无状态**：API/Admin/Worker 节点可水平扩容，状态全部下沉到 MySQL/Redis

## 4. 二进制与模块全景

### 4.1 顶层二进制（cmd/*）
| 二进制 | 默认端口 | 职责 |
|--------|---------|------|
| `cmd/api` | :8080 | 全部 v1 API（含用户面 + admin 路由 + `/sub/:token`），向后兼容入口 |
| `cmd/admin` | :8081 | 运营后台独立站点（嵌入 `web/admin/dist`，只挂 admin 路由） — Phase 11 进行中 |
| `cmd/user-web` | :8082 | 用户控制台独立站点（嵌入 `web/user/dist`） — ✅ Phase 12 已上线 |
| `cmd/worker` | — | Asynq 消费者：邮件、订单超时、流量重置、节点心跳清理、报表生成 |
| `cmd/node-agent` | — | 部署在每台 VPS：拉取配置 / 推流量 / 热重载 Xray |
| `cmd/usdt-watcher` | — | 独立进程：轮询 TronGrid，匹配收款，回写订单 |
| `cmd/seed-admin` | — | 开发工具：一键创建管理员账号 |

### 4.2 业务上下文（internal/* — DDD 分层）
每个上下文遵循统一目录约定：
```
internal/<bounded-context>/
  ├── domain/         # 实体、值对象、领域服务
  ├── ports/          # 仓储接口（依赖反转）
  ├── service/        # 应用服务（用例编排）
  ├── infra/gormrepo  # 仓储实现（GORM）
  └── transport/httpapi  # HTTP handler + 路由 Register
```

| 上下文 | 主要职责 |
|--------|---------|
| `user` | 注册 / 登录 / JWT / 邮箱验证码 / 2FA(TOTP) / 邀请关系 / 余额 / Admin 用户管理 |
| `billing` | 套餐 / 流量包 / 优惠券 / 订单状态机 / 余额账本 / Admin CRUD |
| `payment` | 支付宝 / 微信 / USDT-TRC20 / Mock 支付 / 回调验签 / 幂等 |
| `node` | 节点 + 节点分组 + 健康检测 + 协议配置（Xray/Sing-box/Hy2） |
| `subscription` | Clash / Sing-box / 通用 base64 订阅生成；token 可重置 |
| `traffic` | 流量统计 / 配额校验 / 超额封禁 / 月度重置 |
| `report` | 收入 / 流量 / 订单 / Dashboard 报表（ClickHouse 物化视图） |

### 4.3 公共能力（internal/pkg/*）
- `auth`：JWT issue / verify，refresh token 黑名单（Redis）
- `audit`：管理操作审计落库（`admin_audit_logs`）
- `httpx`：响应信封 `{code, message, data}`、错误码统一、limit/offset 工具
- `middleware`：CORS、Recovery、RequestID、限流（Redis token bucket）、RequireAuth、RequireRole
- `asynqx`：Asynq Server/Client wrapper、任务注册
- `logger`：zerolog 配置，OTel trace ID 注入
- `config`：Viper 加载、环境变量映射
- `idgen`：订单号 / 支付号 / TraceID 生成器
- `storage`：MySQL / Redis / ClickHouse 连接工厂
- `testsupport`：testcontainers helper（MySQL/Redis）

## 6. 核心业务流程

### 6.1 用户购买 → 使用（已 E2E 走通）
1. 注册：发验证码 → 校验 → 创建 user + 唯一 sub_token + 邀请关系绑定
2. 浏览套餐 → 创建订单 `POST /orders {type, target_id, pay_method, coupon_code?}`
3. 发起支付：
   - 支付宝/微信：调起第三方拿二维码/跳转 URL
   - USDT：分配唯一收款地址 + 唯一金额尾数（防混淆）
   - Mock：测试通道，立即结清
4. 回调成功（或 watcher 命中）→ payment.status=paid → 触发 order 服务回写：
   - 流量加成、套餐到期延长、余额到账（topup 类型）
5. 客户端导入订阅 `GET /sub/:token?fmt=clash|singbox`，按用户 plan + node_group 动态生成
6. 客户端连接节点 → node-agent 校验 UUID + 流量配额 → 放行
7. 节点定时上报流量 → traffic 服务累加 → 超额自动封禁 + 邮件提醒

### 6.2 节点纳管
1. 新 VPS 一键脚本部署 node-agent（`deploy/scripts/install-node.sh`）
2. 携带注册 token 上报 `node-svc` → 管理员审核 / 自动入组
3. 心跳 30s；连续 3 次失败 → 自动从订阅剔除
4. 配置热更（用户列表 / 协议参数变更）通过 SIGHUP 或 gRPC 推送

### 6.3 USDT 支付
1. 用户选 USDT-TRC20 → 生成收款地址 + 金额（精确到 6 位小数防撞）
2. `cmd/usdt-watcher` 独立进程轮询 TronGrid
3. 命中（金额 + 地址匹配）→ 调用 payment 服务回写 → 订单已支付
4. 15 分钟未到账自动取消

## 7. 安全设计

| 层面 | 措施 |
|------|------|
| 传输 | 全站 TLS 1.3、HSTS、Cloudflare 前置 |
| 协议反审查 | Reality / Vision / XHTTP；uTLS 指纹伪装 |
| 鉴权 | 短期 JWT（15min）+ Refresh Token（Redis 黑名单）+ TOTP 2FA |
| 支付幂等 | 订单号唯一索引、`channel_trade_no` 唯一、金额二次校验 |
| 节点通信 | gRPC + mTLS + 节点 token 双向认证 |
| 后台 | RBAC（user/admin/ops/finance）+ admin 操作全链路审计 + IP 白名单 |
| 防滥用 | 设备数限制、共享检测、IP 限频、订阅拉取限速 |
| 隐私 | 不记录用户访问目标，仅记录字节数；订阅 token / UUID 可一键轮换 |
| 依赖 | gosec + govulncheck CI；Distroless 镜像；非 root |

## 8. 部署架构

### 8.1 控制面（管理端）
- 单区域部署（推荐日本东京）
- Docker Compose 启 1 套：`api` / `admin-web` / `user-web` / `worker` / `usdt-watcher` / MySQL / Redis / Caddy / Prometheus / Grafana / Loki
- 数据库每日自动备份到 S3；按 14 天滚动

### 8.2 数据面（代理节点）
- 每台 VPS systemd 管理：`xray.service` + `node-agent.service`
- 启用 BBR / BBRv3
- IPv4 + IPv6 双栈
- Cloudflare 中转节点 vs 直连节点两套配置模板

## 11. 风险与合规

1. **法律主体**：注册在合规司法管辖区（HK / SG / US LLC）；ToS 明确禁止违法用途；用户须满 18 岁
2. **支付**：避免国内对公账户；优先海外收单 + USDT
3. **抗审查**：核心域名 + 备用域名双备份；订阅域名与节点域名分离；支持用户自定义订阅域
4. **DDoS**：Cloudflare + 海外高防 IP；管理面与代理面完全隔离
5. **数据隐私**：最小化原则；明示隐私政策；用户可一键删除账户
6. **依赖供应链**：gosec + govulncheck CI；Renovate 自动 PR；二进制带 SBOM
