# 企业级商用 VPN（机场）整体服务框架

> 项目代号：**proxy_VPN**
> 定位：面向中高端用户的商用级多协议代理服务（"机场"），覆盖美国、日本等多地区 VPS 节点，支持套餐 + 流量包混合计费，支持支付宝 / 微信 / USDT 支付。
> 技术总览：Go 后端 + 多协议代理（Xray / Sing-box / Hysteria2）+ 微服务化部署。

---

## 1. 产品定位与目标

| 维度 | 说明 |
|------|------|
| 目标用户 | 个人付费用户、海外留学/工作华人、跨境电商、企业内网代理 |
| 节点分布 | 美国（西海岸/东海岸）、日本（东京/大阪）、可扩展到新/港/台/欧 |
| 核心卖点 | 高可用、高隐私、多协议、低延迟、可视化、可扩展 |
| 商业模式 | 月付/年付套餐 + 流量包叠加 + 邀请返佣 |
| 合规性 | 海外注册主体运营、KYC 可选、日志最小化策略 |

## 2. 技术栈选型

### 2.1 后端
- **语言**：Go 1.22+（高并发、低内存、便于跨平台部署）
- **Web 框架**：Gin / Echo（API 服务），gRPC（内部服务间通信）
- **ORM**：GORM / Ent
- **任务调度**：Asynq（基于 Redis 的任务队列）
- **配置中心**：Viper + Nacos / Consul（可选）

### 2.2 代理核心
- **Xray-core**：VMess、VLESS、Trojan、Shadowsocks、Shadowsocks-2022、Reality
- **Sing-box**：作为统一抽象层，便于协议聚合
- **Hysteria2 / TUIC v5**：基于 QUIC 的高速协议
- **WireGuard**：可选 VPN 模式
- **传输层**：TCP / WebSocket / gRPC / HTTPUpgrade / Reality / XHTTP

### 2.3 数据存储
- **MySQL 8.0**：业务主数据（用户、订单、套餐、节点）
- **Redis 7**：缓存、会话、限流、流量计数、任务队列
- **ClickHouse**：流量日志与统计分析（按需）
- **对象存储**：S3 / R2（订阅文件、客户端分发包）

### 2.4 前端
- **用户端**：Next.js 14（React + TS）+ Tailwind + shadcn/ui
- **管理后台**：Vue3 + Vite + Element Plus（或 Ant Design Pro）
- **移动端**：复用 Web（H5 自适应）；客户端复用社区客户端（v2rayN、Shadowrocket、Sing-box、Stash 等）

### 2.5 基础设施
- **容器**：Docker + Docker Compose（小规模）/ Kubernetes（中大规模）
- **网关**：Nginx / Caddy（自动 TLS）+ Cloudflare CDN 前置
- **监控**：Prometheus + Grafana + Loki + Alertmanager
- **链路追踪**：OpenTelemetry + Jaeger
- **CI/CD**：GitHub Actions + GHCR

## 4. 模块拆分（微服务）

| 模块 | 主要职责 | 关键接口 |
|------|---------|---------|
| **user-svc** | 注册/登录/JWT/2FA/邀请关系/工单 | `/api/v1/auth/*`、`/api/v1/user/*` |
| **plan-svc** | 套餐管理、流量包、价格策略、优惠券 | `/api/v1/plans`、`/api/v1/coupons` |
| **order-svc** | 下单、订单状态机、对账 | `/api/v1/orders` |
| **billing-svc** | 支付宝 / 微信 / USDT (TRC20/ERC20) 接入、回调验签、退款 | `/api/v1/pay/*`、`/callback/*` |
| **node-svc** | 节点注册、健康检测、负载均衡、地区分组 | `/api/v1/nodes`（管理）+ gRPC |
| **subscription-svc** | 生成订阅链接（Clash/Sing-box/Surge/v2rayN 多格式） | `/sub/:token` |
| **traffic-svc** | 流量统计、配额校验、超额停服、月度重置 | gRPC（节点上报） |
| **notification-svc** | 邮件/Telegram/微信公众号通知 | 内部异步 |
| **admin-svc** | 后台管理 API（用户、订单、节点、财务报表） | `/admin/*` |
| **node-agent**（部署在每台 VPS） | 拉取配置、推流量、热重载 Xray | gRPC client |

## 6. 关键业务流程

### 6.1 用户购买 → 使用流程
1. 注册 → 邮箱验证 → 自动生成唯一 `uuid`（用于代理鉴权）
2. 浏览套餐 → 下单 → 选择支付方式
3. 调起支付（支付宝/微信扫码，USDT 链上转账 + 自动确认）
4. 回调成功 → `order-svc` 写入 → `user-svc` 更新套餐 / 流量配额
5. 用户获取订阅链接（含 token）→ 客户端导入
6. `subscription-svc` 按用户套餐 + 节点分组，动态生成 Clash/Sing-box 配置
7. 客户端连接节点 → `node-agent` 校验用户 UUID + 配额 → 放行
8. 节点定时上报流量 → `traffic-svc` 累加 → 超额自动断流

### 6.2 节点纳管流程
1. 新 VPS 部署 `node-agent`（一键脚本：`curl … | bash`）
2. 注册 token 上报到 `node-svc` → 管理员审核
3. 下发 Xray 配置（含 Reality 公钥、ShortID、用户列表）
4. 心跳 30s 一次；连续 3 次失败 → 自动从订阅剔除
5. 支持灰度发布（按节点分组）

### 6.3 USDT 支付流程
1. 用户选择 USDT-TRC20 → 系统生成唯一收款地址 + 唯一金额尾数（防混淆）
2. 后台 `usdt-watcher`（独立服务）轮询 TronGrid / BSC RPC
3. 匹配金额 + 地址 → 标记订单已支付
4. 15 分钟未到账自动取消

## 7. 安全设计

| 层面 | 措施 |
|------|------|
| 传输 | 全站 TLS 1.3、HSTS、Cloudflare 前置防 DDoS |
| 协议反审查 | Reality / Vision / XHTTP，TLS 指纹伪装 (uTLS) |
| 鉴权 | JWT (短期) + Refresh Token (Redis 黑名单)、双因子 (TOTP) |
| 支付 | 回调签名验证、幂等 `trade_no`、金额二次校验 |
| 节点通信 | mTLS + 节点 token，gRPC 双向认证 |
| 日志策略 | 不记录用户访问目标，仅记录流量字节数（合规与隐私平衡） |
| 防滥用 | 设备数限制、共享检测（多 IP 并发告警）、IP 限频 |
| 后台 | RBAC 权限模型、操作审计、IP 白名单 |
| 客户端鉴权 | 订阅链接 token 可重置；UUID 可一键轮换 |

## 8. 部署架构

### 8.1 控制面（管理端）
- 单区域部署（建议日本东京，国内访问较快且海外合规）
- Docker Compose 起 1 套：`api`、`admin-web`、`user-web`、`mysql`、`redis`、`clickhouse`、`prometheus`、`grafana`、`caddy`
- 数据库每日自动备份到 S3

### 8.2 数据面（代理节点）
- 每台 VPS 独立 systemd 管理：`xray.service` + `node-agent.service`
- BBR / BBRv3 拥塞控制开启
- IPv4 / IPv6 双栈
- 通过 Cloudflare 中转或直连两套配置（按节点角色）

### 8.3 推荐目录结构
```

├── cmd/
```

## 11. 风险与合规

1. **法律**：主体注册在合规司法管辖区（香港/新加坡/美国 LLC），ToS 明确禁止违法用途。
2. **支付**：避免使用国内对公账户接收 VPN 收入，建议海外收款 + USDT。
3. **抗审查**：核心域名 + 备用域名双备份；订阅域名与节点域名分离；支持自定义订阅域名。
4. **DDoS**：Cloudflare + 海外高防 IP；管理面与代理面完全隔离。
5. **数据隐私**：遵循最小化原则，明示隐私政策。
